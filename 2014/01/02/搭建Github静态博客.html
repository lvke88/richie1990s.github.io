搭建Github静态博客

为什么选择GitHub Pages

  • 免费，自由，也是一种趋势
  • 团队协作流程和技术大牛，有利于开阔眼界

准备软件

下载并安装以下软件

  1. Node.js
  2. Git

在选定的目录中创建文件夹(此文件夹就是博客的根目录，所有操作都在里面执行，慎重选择)。进入文件夹，在空白处右键选择 Git Bash Here 在弹出的窗口中依次执行下列命令：

//~ 安装Hexo
npm install-g hexo
//~ 初始化Hexo
hexo init

到这里本地博客已经搭建完成，执行下列命令查看：

//~ 生成静态页面
hexo generate (简写: hexo g)
//~ 启动本地服务，进行文章预览调试
hexo server (简写: hexo s)

或者直接输入

//~ 生成静态页面，并启动本地服务，进行文章预览调试
hexo server -g

浏览器输入 http://localhost:4000 即可预览博客网页了。

提示

出现错误
ERROR Deployer not found: git 或者 ERROR Deployer not found: github
解决方法
npm install hexo-deployer-git –save

注册和配置Github

  1. 注册并登录
    进入 Github官网 按照步骤进行注册，注册成功后登录。
  2. 创建仓库
    在首页点击绿色按钮 New repository ,Repository name规则为 用户名.github.io ,其他的默认设置就好了。
  3. 生成博客网页
    进入创建好的仓库，找到 Settings 并点击进入，点击 Automatic page generator 然后填写博客的相关信息并选择博客主题
  4. 验证
    在浏览器中输入 用户名.github.io， 就可以看到你的博客主页了。

配置SSH Keys

  1. 检查本机的SSH密钥，

    cd ~/.ssh

    提示：No such file or directory。继续下面步骤，否则就不用再次生成了。

  2. 生成SSH Key

    ssh-keygen -t rsa -C “邮件地址”
    Generating public/private rsa key pair.
    Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

    然后会要求输入密码，此处的密码是提交项目时使用的，如果为空则不用输入。

    Enter passphrase (empty for no passphrase):<输入加密串>
    Enter same passphrase again:<再次输入加密串>

  3. 添加SSH Key
    打开 <用户目录>/.ssh/id_rsa.pub 复制内容，进入Github，点击右上角的 Account Settings—>SSH Public keys —> add another public keys，将复制的内容粘贴进去就好了。

  4. 测试

    ssh -T git@github.com

    如果是下面的反馈

    The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
    RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
    Are you sure you want to continue connecting (yes/no)?

    输入 yes 就好了

购买域名并绑定Github Pages

  1. 创建CNAME文件
    在仓库的根目录中创建文件CNAME 里面输入你的仓库名(用户名.github.io)，如blog.shulq.com
  2. 解析域名
    创建新的记录
    主机记录 www 记录类型 CNAME 记录值 用户名.github.io
    主机记录 @ 记录类型 A 记录值 192.30.252.153
    主机记录 @ 记录类型 A 记录值 192.30.252.154
